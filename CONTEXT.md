# UPS Battery Monitor — Project Context

## Железо и окружение

- **UPS**: CyberPower UT850EG (425W номинал), подключён по USB
- **Драйвер**: NUT `usbhid-ups` v2.8.1
- **Сервер**: senbonzakura, Debian 13, нагрузка на UPS обычно 16–21%
- **Телеметрия NUT**: `upsc cyberpower@localhost`

## Что мы знаем о поведении UPS

### Реальный блекаут 2026-03-12

- Блекаут начался: **17:53** (зафиксировано в `journalctl -u nut-monitor`)
- Сервер выключился: ~**18:40** (~47 минут на батарее)
- Нагрузка: стабильно **16–18%**
- `battery.charge` упало до **0%** примерно на **35-й минуте** — то есть ещё ~12 минут UPS работал при показании 0%
- `battery.runtime` показывал 0 сек при этом же

### Почему автошатдаун не сработал

NUT не объявил critical state, потому что в конфиге драйвера стоит `onlinedischarge_calibration: true`. При OB+LB+DISCHRG NUT решил, что это калибровочный прогон, и не инициировал shutdown.

Лог:
```
is_ups_critical: UPS [cyberpower] is OB+LB now, but it is also calibrating - not declaring a critical state
```

### Почему калибровка прошивки не помогла

После полного разряда и перезарядки `battery.runtime` остался ~1300–1400 сек (~22 мин) — те же значения что до блекаута. Проверили `upscmd -l`: команды `calibrate.start` нет. Прошивка UT850EG не поддерживает пересчёт ёмкости через NUT.

## Бизнес-требования

### 1. Точный прогноз остаточного времени работы

**Проблема**: NUT врёт в ~2 раза. При нагрузке 16–18% реальное время ~45–50 мин, NUT говорит ~22 мин.

**Решение**: строить собственную модель на основе `battery.voltage` + `ups.load`.

- `battery.voltage` — физически надёжный показатель разряда PbAcid
- Кривая разряда предсказуема и воспроизводима
- Поправка на нагрузку: `remaining_time ∝ 1 / load`

**Точность**: достаточная для цели — предупредить за 2–3 минуты до смерти.

### 2. Безопасный shutdown

При предсказанном остатке ≤ N минут (например, 3 мин) — инициировать `sudo shutdown -h +2`.

Это решает проблему с `onlinedischarge_calibration`: не полагаемся на NUT critical state, работаем по своей модели.

### 3. Деградация батареи

Сравнивать суммарную ёмкость (площадь под кривой voltage×time) от цикла к циклу. Батарея деградирует — площадь уменьшается.

### 4. Алерт "пора менять батарею"

**Триггер**: если при 100% заряде и средней нагрузке за последние 30 дней расчётное время работы < 15 минут → алерт.

Алерт в MOTD + (опционально) Telegram через mcp-telegram.

## Архитектура (предварительная)

### Компонент 1: Сборщик телеметрии

- Пишет каждые 30 сек: `timestamp, voltage, load, charge_nut, runtime_nut, ups_status`
- Хранит всегда (не только при блекауте)
- Хранилище: SQLite или CSV — **TBD**

### Компонент 2: Калибровщик

- Детектирует полный цикл разряда (OB → восстановление питания или выключение)
- Строит кривую `voltage → remaining_seconds` при эталонной нагрузке
- Интерполяция по реальным точкам
- Сохраняет модель (JSON с таблицей точек)
- Источник данных для калибровки: реальные блекауты или ручной тест (выдернуть вилку)

### Компонент 3: Монитор / предиктор

- При `OB DISCHRG` запускается в активный режим (опрос каждые 30 сек)
- По текущему `voltage` + `load` смотрит в калиброванную модель
- Пишет предсказанный остаток в `/run/nut/predicted_runtime`
- При достижении порога — инициирует shutdown

### Компонент 4: Отслеживание деградации

- После каждого цикла сохраняет суммарную ёмкость
- Строит тренд деградации
- Алерт при падении ниже порога
