  <name>Task 1: Integrate write_virtual_ups_dev() call into monitor.py polling loop</name>
  <files>src/monitor.py</files>
  <action>
    Extend monitor.py to construct virtual_metrics dict and write to tmpfs in the main polling loop.

    Location: In monitor.py.run() method, within the polling loop (line 208-275 area where metrics are logged).

    Implementation:
    1. After all calculations are complete (soc, battery_charge, time_rem computed and stored in self.current_metrics):
       - Build virtual_metrics dict with:
         * Override fields: battery.runtime (time_rem * 60 to convert to seconds), battery.charge (as %), ups.status (from compute_ups_status_override call)
         * Passthrough fields: All other keys from ups_data except the 3 overrides
       - Example construction:
         ```python
         virtual_metrics = {
             "battery.runtime": int(time_rem * 60),  # seconds
             "battery.charge": int(battery_charge) if battery_charge is not None else 0,
             "ups.status": ups_status_override,
             **{k: v for k, v in ups_data.items()
                if k not in ["battery.runtime", "battery.charge", "ups.status"]}
         }
         ```

    2. Compute ups.status override before constructing virtual_metrics:
       - Call compute_ups_status_override(event_type, time_rem, SHUTDOWN_THRESHOLD_MINUTES)
       - Store in ups_status_override variable
       - Use in virtual_metrics dict above

    3. Call write_virtual_ups_dev(virtual_metrics) after metrics dict is built
       - Place after logging (line 273) so log includes the write
       - Wrap in try/except to catch any tmpfs I/O errors
       - Log any exceptions but DON'T crash daemon (just continue polling)

    4. Error handling:
       - Catch Exception (write_virtual_ups_dev will raise on serious errors)
       - Log: logger.error(f"Failed to write virtual UPS metrics: {e}")
       - Continue polling (don't break out of loop)
       - Optionally add counter for consecutive write failures (warning if > 3 in a row?)

    Code structure (pseudocode):
    ```python
    # After line 273 metrics logging
    try:
        # Compute status override
        ups_status_override = compute_ups_status_override(
            event_type,
            time_rem,
            SHUTDOWN_THRESHOLD_MINUTES
        )

        # Build virtual metrics dict
        virtual_metrics = {
            "battery.runtime": int(time_rem * 60) if time_rem is not None else 0,
            "battery.charge": int(battery_charge) if battery_charge is not None else 0,
            "ups.status": ups_status_override,
            **{k: v for k, v in ups_data.items()
               if k not in ["battery.runtime", "battery.charge", "ups.status"]}
        }

        # Write to tmpfs
        write_virtual_ups_dev(virtual_metrics)
    except Exception as e:
        logger.error(f"Failed to write virtual UPS metrics: {e}")
    ```

    Import statement:
    - Add at top: from src.virtual_ups import write_virtual_ups_dev, compute_ups_status_override

    Frequency: Write happens every poll cycle (every POLL_INTERVAL seconds, typically 10 sec).
    - This is intentional: NUT dummy-once mode re-reads on timestamp change; frequent updates ensure fresh data
