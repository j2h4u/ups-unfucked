    ```python
    def soc_from_voltage(v_norm: float, lut: List[Dict[str, float]]) -> float:
        """
        Map normalized voltage to SoC using LUT linear interpolation.

        Args:
            v_norm: IR-compensated voltage (volts)
            lut: List of {"v": voltage, "soc": state_of_charge, "source": "..."} dicts

        Returns:
            SoC as fraction [0.0, 1.0]
        """
    ```

    Implementation from 02-RESEARCH.md § Pattern 1:
    1. Extract voltage column from LUT: `voltages = [entry['v'] for entry in lut]`
    2. Handle edge cases:
       - If v_norm >= max voltage: return max SoC (1.0)
       - If v_norm <= min voltage (anchor): return min SoC (0.0)
    3. Use `bisect.bisect_left()` to find insertion point: find two adjacent LUT points that bracket v_norm
    4. Linear interpolation: `SoC = SoC_low + (v_norm - v_low) / (v_high - v_low) * (SoC_high - SoC_low)`
    5. Clamp result to [0.0, 1.0]

    Import:
    ```python
    import bisect
    from typing import List, Dict
    ```

    **Why this approach:**
    - `bisect` is O(log N) efficient for sorted voltage lists
    - Linear interpolation is sufficient for measured VRLA data (not smooth function)
    - Clamping prevents nonsensical SoC values outside [0, 1]

    Reference implementation verified in 02-RESEARCH.md code example (lines 387-419).

    ```python
    def charge_percentage(soc: float) -> float:
        # Clamp SoC to [0, 1] first
        soc_clamped = max(0.0, min(1.0, soc))
        # Convert to percentage
        return soc_clamped * 100.0
    ```

    **Rationale:** Simple direct conversion. Clamping guards against upstream arithmetic errors.
