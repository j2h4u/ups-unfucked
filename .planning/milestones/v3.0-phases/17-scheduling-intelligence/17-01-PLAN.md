**Guard clauses (enforcement order):**
1. SoH floor gate (SCHED-05): if `soh_percent < soh_floor_threshold` → `block_test` reason="soh_floor_X%"
2. Rate limiting gate: if `days_since_last_test < min_days_between_tests` → `defer_test` reason="rate_limit_Xd_remaining"
3. Blackout credit gate (SCHED-03): if `active_blackout_credit.active` and not expired → `defer_test` reason="blackout_credit_active_until_DATE"
4. Grid stability gate (SCHED-06, **CONFIGURABLE**):
   - if `grid_stability_cooldown_hours == 0`: skip this gate entirely (disabled)
   - else if `grid_stability_cooldown_hours > 0` and `last_blackout_timestamp` within last N hours → `defer_test` reason="grid_unstable_blackout_Xh_ago"
5. Cycle budget gate: if `cycle_budget_remaining < 5` → `block_test` reason="critical_cycle_budget"
6. ROI gate (marginal benefit): if `roi < roi_threshold` and `cycle_budget > 20` → `defer_test` reason="marginal_roi_X"
7. Sulfation threshold:
   - if `sulfation_score > 0.65` → `propose_test` test_type='deep' reason="sulfation_X_roi_Y"
   - elif `sulfation_score > 0.40` → `propose_test` test_type='quick' reason="sulfation_X_ir_measure"
   - else → `defer_test` reason="low_sulfation_X" next_eligible=now+2days

**Key implementation rules:**
- No I/O, no logging, no side effects
- datetime handling: use `datetime.now(timezone.utc)` for all comparisons
- Return reason_code that describes decision: "soh_floor_55%", "rate_limit_3d_remaining", "blackout_credit_active", etc.
- set next_eligible_timestamp for defer/block decisions (helps executor predict when test becomes eligible)

```python
def _calculate_days_since_last_test(self) -> float:
    """Return days since last upscmd, or inf if never tested."""
    last_ts = self.battery_model.data.get('last_upscmd_timestamp')
    if not last_ts:
        return float('inf')
    try:
        last_dt = datetime.fromisoformat(last_ts)
        return (datetime.now(timezone.utc) - last_dt).total_seconds() / 86400.0
    except ValueError:
        return float('inf')

def _get_last_natural_blackout(self) -> dict | None:
    """Return most recent natural blackout event (DoD, timestamp)."""
    events = self.battery_model.data.get('discharge_events', [])
    for event in reversed(events):  # Most recent first
        if event.get('event_reason') == 'natural':
            return {
                'timestamp': event.get('timestamp'),
                'depth': event.get('depth_of_discharge', 0.0),
            }
    return None
```

**2. Test-initiated vs natural classification** (in event_classifier or discharge_handler):

```python
def _classify_discharge_event(self, discharge_data: dict) -> str:
    """
    Classify discharge as 'test_initiated' or 'natural'.

    Heuristic: If discharge starts within 60 seconds of upscmd timestamp,
    it's test-initiated. Otherwise, natural.
    """
    last_upscmd = self.battery_model.data.get('last_upscmd_timestamp')
    discharge_start = discharge_data.get('timestamp')

    if not last_upscmd or not discharge_start:
        return 'natural'  # No upscmd record, assume natural

    try:
        upscmd_dt = datetime.fromisoformat(last_upscmd)
        discharge_dt = datetime.fromisoformat(discharge_start)
        time_delta = (discharge_dt - upscmd_dt).total_seconds()

        # If discharge started within 60s of upscmd, it's test-initiated
        if 0 <= time_delta <= 60:
```
