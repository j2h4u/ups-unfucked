# Phase 12.1 Plan 02: Year-Long Simulation Harness Summary

**Wave 2 deliverable:** Year simulation infrastructure for battery_math kernel stability testing.

## Objective

Build a year-long simulation harness that iterates battery_math kernel functions through 365 days of synthetic discharge events. Simulation is orchestrator-agnostic (no MonitorDaemon, no model.json I/O) — purely functional: BatteryState in, events applied, new BatteryState out.

**Purpose:** Enables deterministic stress-testing of kernel functions; validates mathematical properties (convergence, stability) before Phase 12 adds capacity estimation.

## Key Design Decisions

### 1. 10 Voltage Samples per Synthetic Discharge
- **Trade-off:** Accuracy vs performance
- **Rationale:** 10 samples sufficient for SoH calculation (trapezoidal rule AUC), enables 365-day year in <30ms
- **Evidence:** Full year with ~700 events, 10 samples each = 7000 AUC calculations in 0.02s = 3.5ms per year
- **Impact:** Wave 3+ stability tests can iterate fast enough for interactive development

### 2. Linear Voltage Interpolation
- **Choice:** Linear v(t) during discharge (vs quadratic or cubic models)
- **Rationale:** Sufficient for synthetic testing; real discharge curves are piecewise linear anyway
- **Trade-off:** Simplicity (faster to compute) vs accuracy (real curves have knee region)
- **Impact:** Stable, deterministic, no special cases

### 3. Immutable BatteryState via frozen=True
- **Design:** frozenstate dataclass prevents accidental mutation
- **Pattern:** Every update via `replace()` creates new state, making circular dependencies visible
- **Enforcement:** Tests verify FrozenInstanceError on attempted mutation
- **Impact:** Kernel functions are pure (BatteryState in, BatteryState out)

### 4. Parametrized Random Seeds
- **Seeds:** [42, 123, 456, 789, 999] — 5 distinct RNG sequences
- **Rationale:** Enables reproducibility testing without mocking time.time()
- **Impact:** Wave 3 stability tests can parallelize across seeds

### 5. Blackout Frequency Distribution (1–5 events/week)
- **Model:** 60% chance of 0–2 events/day, 40% chance of 2–5 events/day
- **Expected:** ~1000 events over 365 days (realistic battery stress)
- **Evidence:** Observed 686–784 events per run (seed variance is expected)
- **Rationale:** Matches observed frequency from 2026-01 field data (see MEMORY.md: partial blackouts 2–5 times/week)
