## Tasks Completed

### Task 1: Refactor monitor.py._auto_calibrate_peukert() to call kernel function

**What was done:**
- Added import: `from src.battery_math import calibrate_peukert`
- Refactored `_auto_calibrate_peukert()` method:
  - Guard clauses (orchestrator responsibility):
    - `len(times) < 2`: Minimum sample count
    - `discharge_duration < 60`: Minimum discharge duration (60s)
    - `avg_load <= 0 or avg_load > 100`: Valid load range validation
  - Call pure kernel function with 7 arguments:
    - `actual_duration_sec`: From discharge buffer times
    - `avg_load_percent`: Computed from `discharge_buffer.loads`, not EMA
    - `current_soh`: SoH value calculated in `_update_battery_health`
    - `capacity_ah`: **RATED capacity from `self.config.capacity_ah`** (VAL-02 constraint)
    - `current_exponent`: Current Peukert exponent from model
    - `nominal_voltage`: From battery model config
    - `nominal_power_watts`: From battery model config
  - Handle kernel result: Apply if not None, log error if None

**Why this matters:**
- Clean separation of concerns: orchestrator handles policy (when to calibrate), kernel handles physics (how to calibrate)
- Enables testing kernel independently without daemon state
- VAL-02 constraint enforced: rated capacity prevents mixing capacity measurements into calibration during Phase 12 estimation
