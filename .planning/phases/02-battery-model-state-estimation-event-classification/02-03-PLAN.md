    ```python
    def runtime_minutes(soc: float, load_ema: float, capacity_ah: float = 7.2,
                        soh: float = 1.0, peukert_exp: float = 1.2, const: float = 215) -> float:
        # Guard against edge cases
        if load_ema <= 0 or soc <= 0:
            return 0.0

        # Peukert: Time = (Ah * SoC * SoH) / (Load ^ Peukert_exp) * Const
        time_rem = (capacity_ah * soc * soh) / (load_ema ** peukert_exp) * const

        return max(0.0, time_rem)
    ```

    **Critical tuning note:**
    - `const = 215` is empirically derived from 2026-03-12 blackout:
      - Observed: 47 minutes at ~20% load with SoC=1.0, SoH=1.0
      - Solve: 47 = (7.2 * 1.0 * 1.0) / (20 ^ 1.2) * Const
      - Result: Const ≈ 47 * (20 ^ 1.2) / 7.2 ≈ 215
    - If real blackout duration changes significantly from 47 min, this constant must be recalibrated
    - Make const configurable via environment variable for future tuning: `UPS_MONITOR_PEUKERT_CONST`

    **Why edge case guards:**
    - load_ema ≤ 0: Prevents division by zero in exponential (0^1.2 = 0, could cause division by zero)
    - soc ≤ 0: No battery available; return 0 immediately rather than negative time
