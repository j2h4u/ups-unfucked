```python
class SagTracker:
    """Voltage sag state machine and ir_k auto-calibration.

    Measures voltage sag on OL->OB transitions to estimate battery internal
    resistance. Owns the IDLE -> MEASURING -> COMPLETE state machine and
    the RLS estimator for ir_k calibration.
    """
```

**Constructor** `__init__(self, battery_model: BatteryModel, rls_ir_k: ScalarRLS, ir_k: float)`:
- Store `self.battery_model`, `self.rls_ir_k`, `self.ir_k`
- Initialize state: `self._state = SagState.IDLE`, `self._v_before_sag = None`, `self._sag_buffer = []`
- Import `SagState`, `SAG_SAMPLES_REQUIRED` from `src.monitor_config`

**Public interface:**
- `track(self, voltage: float, event_type, transition_occurred: bool, current_load: float | None)` -- drives the state machine. Uses `event_type` and `transition_occurred` (instead of reading from `self.event_classifier` like the old code). Uses `current_load` (instead of reading `self.ema_filter.load`). On OL->OB transition (transition_occurred=True AND event_type not ONLINE): capture `voltage` as v_before_sag (this is the pre-sag EMA voltage, passed by the caller who reads it before the sag develops), reset buffer, set MEASURING. On OB->OL (transition_occurred=True AND event_type==ONLINE): cancel if MEASURING. During MEASURING: append voltage, check buffer length >= SAG_SAMPLES_REQUIRED, compute median of last 3, call _record_voltage_sag.
- `is_measuring` property -> bool: returns `self._state == SagState.MEASURING`
- `reset_idle(self)` -> None: sets `self._state = SagState.IDLE`
- `reset_rls(self, theta: float, P: float)` -> None: creates fresh `ScalarRLS(theta=theta, P=P)`, sets `self.ir_k = theta`, assigns to `self.rls_ir_k`

**Private methods:**
- `_record_voltage_sag(self, v_sag: float, event_type)` -- extracted from MonitorDaemon._record_voltage_sag(). Computes I_actual from `current_load` (stored during track call or passed directly). Uses `self.battery_model.get_nominal_voltage()` and `self.battery_model.get_nominal_power_watts()`. Computes `r_ohm = delta_v / I_actual`, calls `self.battery_model.add_r_internal_entry(...)`. Runs RLS update: `ir_k_measured = r_ohm * nominal_power_watts / (nominal_voltage * 100.0)`, `new_ir_k, new_P = self.rls_ir_k.update(ir_k_measured)`, clamp to [0.005, 0.025], store `self.ir_k = new_ir_k`, call `self.battery_model.set_ir_k(new_ir_k)` and `self.battery_model.set_rls_state('ir_k', new_ir_k, new_P, self.rls_ir_k.sample_count)`. Log both ir_k_calibration and voltage_sag events with structured extras (same format as existing code).

**Design note on current_load for _record_voltage_sag:** The track() method needs `current_load` available when recording. Store it as `self._current_load` during track() so _record_voltage_sag can use it. This is cleaner than passing a callable.

**Design note on v_before_sag:** In the existing code, `self.v_before_sag = self.ema_filter.voltage` captures the EMA voltage at transition time. In SagTracker, the caller (MonitorDaemon._poll_once) will pass the EMA voltage as the `voltage` parameter to track(). On transition to MEASURING, SagTracker stores this as `self._v_before_sag`. The key insight: the first call at transition time gets the pre-sag EMA voltage because EMA hasn't yet incorporated the sag sample.
