## Decision Tree Logic (from 17-RESEARCH.md)

Scheduling gates in order (return early on failure):
1. **SoH floor gate** — if SoH < 60% → `block_test` (hard block, no cycles available)
2. **Rate limiting gate** — if days_since_last_test < 7.0 → `defer_test`
3. **Blackout credit gate** — if active_blackout_credit exists and not expired → `defer_test`
4. **Grid stability gate** — if recent natural blackout in last 24h → `defer_test`
5. **Cycle budget gate** — if cycle_budget < 5 → `block_test` (hard block, critical)
6. **ROI gate** — if roi < 0.2 AND cycle_budget > 20 → `defer_test`
7. **Proposal gate** — if sulfation > 0.65 → `propose_test(deep)`, elif > 0.40 → `propose_test(quick)`, else → `defer_test`

All decisions include `reason_code` string: e.g., "soh_floor_55%", "rate_limit_4d_remaining", "sulfation_0.65_roi_0.34"

## Precondition Checks (before upscmd dispatch)

Called right before `send_instcmd()`, gates are:
1. UPS status not OB or CAL (must be OL to test)
2. SoC ≥ 95% (battery charged)
3. Recent power glitches ≤ 2 in last 4h (grid stable)
4. No test already running (no concurrent tests)

Each check returns `(can_proceed: bool, reason_if_blocked: str)`
