**Signature:**
```python
class CapacityEstimator:
    def __init__(self, peukert_exponent: float, nominal_voltage: float = 12.0,
                 nominal_power_watts: float = 425.0):
        """
        Args:
            peukert_exponent: MUST come from model.get_peukert_exponent(), NOT hardcoded.
                              Callers pass the value; this class does not auto-refine it.
            nominal_voltage: Battery nominal voltage (V)
            nominal_power_watts: UPS rated power (W)
        """
        self.peukert_exponent = peukert_exponent
        self.nominal_voltage = nominal_voltage
        self.nominal_power_watts = nominal_power_watts
        self.measurements = []

    def estimate(
        self,
        voltage_series: List[float],
        time_series: List[float],
        current_series: List[float],
        lut: List[Dict],
    ) -> tuple | None:
        """
        Estimate capacity from a single discharge event.

        Returns:
            (Ah_estimate: float, confidence: float, metadata: dict)
            OR None if quality filter rejects
        """
```

**Core methods (per RESEARCH.md):**
1. `_passes_quality_filter(V, t, I, lut)` → bool
   - Reject if duration < 300s
   - Reject if ΔSoC < 5% (flicker)
   - Reject if ΔSoC < 25% (shallow)
   - Return True only if all pass

2. `_integrate_current(I_percent, t, nominal_power_watts, nominal_voltage)` → float
   - Convert load% → current (A) = (load_percent / 100) × power_watts / voltage
   - Trapezoidal integration: Ah = ∑[(I_i + I_{i+1})/2 × ΔT / 3600]
   - Reference implementation in RESEARCH.md coulomb_integrate()

3. `_get_soc_range(V, lut)` → (soc_start, soc_end)
   - Use existing `soc_from_voltage()` from soc_predictor.py
   - Return (SoC at V[0], SoC at V[-1])

4. `_estimate_from_voltage_curve(V, t, delta_soc)` → float
   - Cross-check coulomb estimate against voltage discharge curve
   - Use Peukert's law reference curve (existing peukert_runtime_hours() pattern)
   - Returns Ah_voltage for outlier rejection (coulomb vs voltage >20% → skip)

5. `_compute_ir(V, I_percent)` → float
   - IR (mΩ) = (V_start - V_end) / I_avg × 1000
   - Expert panel requirement for v3.0 trending
   - Return 0 if I_avg = 0 (avoid division by zero)

6. `_compute_confidence()` → float
   - convergence_score = 1 - CoV (clamped to [0, 1])
   - CoV = population_std(Ah_values) / mean(Ah_values)
   - Use population std (divide by n, NOT n-1) — more stable for small n, this is a UI metric not a statistical test
   - Fewer than 3 measurements: return 0.0 (CoV with 1-2 samples is meaningless)
   - By sample 3 (if CoV < 0.10): score >= 0.90
   - NOTE: score may decrease when a noisy sample is added — this is expected, NOT monotonic

7. `add_measurement(ah, timestamp, metadata)` → None
   - Append (timestamp, ah, confidence, metadata) to self.measurements
   - Update confidence via _compute_confidence() after each add

8. `has_converged()` → bool
   - Return len(measurements) >= 3 AND CoV < 0.10
   - Expert panel approved: "count ≥ 3 AND CoV < 0.10"

**Implementation notes:**
- Use stdlib only: math, statistics, json, datetime (no external deps, per RESEARCH standard stack)
- Import `soc_from_voltage` from src.soc_predictor
- For voltage-curve estimate, reference existing peukert_runtime_hours() pattern (Phase 1 code)
- Store all metadata in dict per measurement for logging (Phase 13 will consume this)
- No logging calls; leave logging to monitor.py (separation of concerns)
- No atomic write in this class; model.py handles persistence (Plan 02)
