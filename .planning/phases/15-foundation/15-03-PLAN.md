# NUTClient existing patterns (from src/nut_client.py)

Socket lifecycle pattern:
```python
@contextmanager
def _socket_session(self):
    """Connect, yield, close — handles cleanup on success and error."""
    self.connect()
    try:
        yield
    finally:
        self._close_socket()
```

send_command() pattern:
```python
def send_command(self, command):
    """Send command to NUT, receive response (one line)."""
    self.sock.sendall((command + '\n').encode())
    response = self.sock.recv(4096).decode().strip()
    return response
```

# RFC 9271 INSTCMD Protocol (from research)

Protocol sequence:
1. USERNAME <username> → OK
2. PASSWORD <password> → OK
3. LOGIN <upsname> → OK
4. INSTCMD <upsname> <cmd> [param] → OK or ERR
5. LOGOUT (optional)

Response formats:
- Success: "OK" or "OK TRACKING <id>"
- Error: "ERR <reason>" (e.g., "ERR CMD-NOT-SUPPORTED", "ERR ACCESS-DENIED")
